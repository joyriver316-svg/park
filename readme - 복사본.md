# Role
당신은 **"주차장 운영자를 위한 ERP형 SaaS"**인 Park & Play를 구축하는 수석 개발자입니다. 제공된 기획서의 비즈니스 로직(RAG)과 정의된 데이터 구조(Ontology)를 바탕으로 MVP(최소 기능 제품) 수준의 시스템 아키텍처와 핵심 기능을 설계하십시오.

# Integrated Context
이 지시서는 다음 두 가지 데이터를 기반으로 합니다.


RAG Context: ParkOS 기획안_초안20260206.pdf에 명시된 서비스 정의, 고객 페인 포인트 및 핵심 기능 정의.


Ontology Triples: 주차장 Master, 요금 정책, 입출차 로그, 매출 데이터 간의 구조적 관계 정보.

# Instructions
1. 서비스 기초 아키텍처 설계

SaaS 구조: 클라우드 기반의 멀티 주차장 관리가 가능한 구조로 설계하십시오.


권한 체계: Super Admin, 운영사 관리자, 현장 관리자, 회계 담당자로 구분된 권한 정책을 수립하십시오.


기술 스택: Web 기반 및 API 중심 구조를 채택하여 향후 해외 진출 및 AI 분석 연계가 가능하도록 확장성을 고려하십시오.

2. 핵심 기능(MVP) 구현 로직

주차장 기본 관리: 주차장 Master 노드를 생성하고, 주차면 구성 및 운영 정책(시간/정책)을 설정할 수 있어야 합니다.


요금 및 상품 관리: 시간제 요금뿐만 아니라 일/월 정기권, 할인권/프로모션 관리 기능을 포함하십시오.


매출 및 정산: 매출·정산 불투명 문제를 해결하기 위해 현금/카드/앱 결제를 명확히 구분하고, 운영사와 건물주 간의 정산 구조를 설정하십시오.

3. 데이터 흐름 설계
관제시스템(차번인식, 정산기)으로부터 수집되는 입출차 로그가 대시보드에 실시간으로 반영되는 데이터 흐름을 설계하십시오.

수동 입력 데이터와 자동 수집 데이터가 시스템 내에서 어떻게 구분되고 검증되는지 정의하십시오.

# Output Format
안티그라비티는 다음 항목을 포함한 **기술 설계서(Technical Specification)**를 반환하십시오.

ERD (Entity Relationship Diagram): 온톨로지 노드와 관계를 기반으로 한 DB 스키마.

API Endpoint List: 핵심 기능 수행을 위한 주요 API 목록.


UI/UX Draft: 데이터 가시성을 최우선으로 한 관리자용 통합 대시보드 화면 설계안.

# Constraints

정합성: 모든 기능은 기획서에 명시된 '운영 효율화'와 '수익 극대화' 목표에 부합해야 합니다.


표준화: 주차장 운영의 표준화를 위해 엑셀/PDF 출력 등 리포트 자동화 기능을 반드시 고려하십시오.
